Motivation, Problem Statement & Objectives

The Gap & Opportunity
  • SQL is a high barrier for domain experts — analysts, managers, planners
  • Multi-join star-schema queries demand specialist technical knowledge
  • LLMs generate accurate SQL from plain natural language
  • Multi-agent pipelines safely decompose complex analytical tasks
Problem Statement
Automatically generate safe, correct SQL from a natural-language question, execute it, discover patterns, and present results — no SQL knowledge required.
Objectives
Architecture
8-agent NL-to-SQL pipeline on a retail star schema
LLM Integration
Qwen2.5-Coder-7B + rule-based offline fallback
SQL Safety
3-layer enforcement — no DML reaches the DB
CSV Ingestion
Auto CSV → star schema in 7 steps
Analytics
Z-score anomaly · trend · chart inference
Export & UI
CSV · XLSX · PDF · Streamlit web app

SQL Generation & Safety Validation

LLM Mode — HuggingFace Inference API
  • Model: Qwen/Qwen2.5-Coder-7B-Instruct
  • Prompt: schema + glossary + NL question + plan + SQLite rules
  • JSON output: sql · chart_hint · tables_used
  • 4-level parse fallback: fence → regex → partial JSON
Rule-Based Fallback (Offline)
PatternTrigger Keywords

3-Layer SQL Safety
1
DML / DDL Keyword Blocklist
INSERT · UPDATE · DELETE · DROP · ALTER · CREATE · GRANT · TRUNCATE · MERGE
2
Dangerous Function Blocklist
pg_sleep · dblink · copy
3
AST-Based SELECT-Only Enforcement
sqlglot parses SQL (SQLite dialect). Root must be SELECT or WITH. Rejects semicolons.
EXPLAIN pre-check — catches plan-time errors before any data is accessed.

Column Role Detection
  • ID  *_id, *_key, *_code
  • Date  datetime dtype or *date*, *time*
  • Measure  numeric + *revenue*, *amount*
  • Dimension  cardinality ≤ min(50, 5%) + hint
Blueprint Output
  • Fact table containing all columns
  • One dim lookup per qualifying categorical column
  • Orchestrator scoped via set_active_blueprint()
  • Built-in warehouse tables hidden from schema context

Chart Type Logic & Pattern Discovery

Pattern Discovery
Anomaly DetectionZ-Score
Flag |z| ≥ 1.75 on first numeric col. Requires ≥ 5 rows.
Trend QuantificationΔ%
Δ% = (vlast−vfirst) / vfirst × 100 · date col + ≥3 rows
Cluster Hintk-means
Suggests k-means / DBSCAN when ≥2 numeric cols + ≥12 rows.
▸ Auto-bucket to monthly when line chart >50 unique x-values
